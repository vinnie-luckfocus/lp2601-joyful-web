# Task 61: Database schema and migrations for videos and highlights

## Description

Create PostgreSQL tables and Prisma migrations for the video replay system: `videos` to store video metadata (linked to `games`) and `video_highlights` to store timestamped highlight markers.

## Acceptance Criteria

- `videos` table exists with fields: `id`, `game_id` (FK), `title`, `description`, `video_url`, `thumbnail_url`, `duration`, `status` (uploading/ready/error), `created_at`, `updated_at`
- `video_highlights` table exists with fields: `id`, `video_id` (FK), `time` (seconds), `description`, `created_at`
- Prisma migration is generated and applies cleanly
- Foreign key constraints and indexes (`game_id`, `video_id`, `status`) are in place
- Seed data includes at least one sample video with highlights for local development

## Technical Details

- Use Prisma schema in `backend/prisma/schema.prisma`
- Run `npx prisma migrate dev --name add_videos_and_highlights`
- Add `onDelete: Cascade` for `video_highlights` when a video is deleted
- Add `@index([game_id])` and `@index([video_id])`

## Dependencies

- Existing `games` table in Prisma schema
- PostgreSQL database reachable locally and in test environments

## Effort Estimate

4 hours

## Definition of Done

- [ ] Migrations generated and committed
- [ ] Schema file updated
- [ ] Seed script updated with sample video + highlights
- [ ] `prisma migrate dev` runs successfully in a fresh environment
