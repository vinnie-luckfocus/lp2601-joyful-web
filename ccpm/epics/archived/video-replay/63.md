# Task 63: Backend upload initialization and completion endpoints

## Description

Implement upload initialization (presigned URL generation) and a completion webhook so the frontend can upload videos directly to object storage and the backend can persist metadata after upload finishes.

## Acceptance Criteria

- `POST /api/videos/upload-init` accepts `{ game_id, file_name, file_size, mime_type }` and returns `{ video_id, presigned_url, expires_at }`
- Creates a `videos` row with `status: uploading` on initialization
- `POST /api/videos/upload-complete` accepts `{ video_id, object_key, duration }` and updates status to `ready`, stores `video_url`, and triggers thumbnail generation (placeholder job acceptable)
- Admin-only authorization enforced on both endpoints
- Handles idempotency: duplicate `upload-complete` calls do not create duplicate records

## Technical Details

- Choose one OSS provider (Aliyun OSS or Tencent COS) based on existing credentials
- Use SDK to generate PUT presigned URLs with expiration ~15 minutes
- Store `object_key` on the video row or derive it from `video_id`
- Add `backend/src/services/videoUpload.ts` for upload logic
- Use existing role-based auth middleware to restrict to admins

## Dependencies

- Task 61 (DB schema)
- Object storage credentials in environment variables
- Existing games API/data to validate `game_id`

## Effort Estimate

8 hours
