# Task 65: Frontend video list page

## Description

Build the video list page with a hero card for the latest game and a responsive grid of historical videos.

## Acceptance Criteria

- Route `/videos` renders a hero card for the most recent video (full width, 16:9 cover, large title and date)
- Historical videos render below in a 2-column grid on desktop and 1-column on mobile
- Each card shows: cover image (16:9), game title/date, home vs away teams, highlights count
- Clicking any card navigates to `/videos/:id`
- Loading and empty states are handled gracefully
- Uses blur-up lazy loading for cover images

## Technical Details

- Use React + Vite + Tailwind CSS per project stack
- Use React Router for navigation
- Fetch from `GET /api/videos?limit=20`
- Store data in a local component state or lightweight Zustand store
- Use `loading="lazy"` and a low-quality placeholder/blur-up technique for images

## Dependencies

- Task 62 (video list API)
- Tailwind and existing UI components (Button, Card, etc.)

## Effort Estimate

8 hours

## Definition of Done

- [ ] `/videos` page implemented with hero + grid layout
- [ ] Responsive grid (2-col desktop, 1-col mobile)
- [ ] Navigation to `/videos/:id` works
- [ ] Loading and empty states implemented
- [ ] Blur-up lazy loading for cover images
- [ ] Unit/component tests pass with >= 80% coverage
- [ ] Visual smoke test in browser completes
