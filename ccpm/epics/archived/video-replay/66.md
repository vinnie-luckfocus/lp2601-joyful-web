# Task 66: Frontend video playback page

## Description

Build the video playback page using the native HTML5 `<video>` element with standard controls and a styled page layout.

## Acceptance Criteria

- Route `/videos/:id` renders a full-width video player with the source from the API
- Player supports play/pause, seek, volume, and playback speed via native controls
- Page displays game title, date, and teams below the player
- Handles loading, error, and 404 states
- Video URL falls back gracefully if the video is still processing (`status: uploading`)

## Technical Details

- Use `video` element with `controls` and `preload="metadata"`
- Fetch from `GET /api/videos/:id`
- Use React Router `useParams` for the video ID
- Display a friendly message when `status !== 'ready'` instead of a broken player

## Dependencies

- Task 62 (video detail API)
- Existing layout/shell components

## Effort Estimate

6 hours

## Definition of Done

- [ ] `/videos/:id` page renders video player
- [ ] Native controls support play/pause/seek/speed
- [ ] Game metadata displayed below player
- [ ] Loading, error, and not-ready states handled
- [ ] Unit/component tests pass with >= 80% coverage
- [ ] Visual smoke test in browser completes
