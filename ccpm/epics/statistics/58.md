# Task 002: Backend API — POST /api/games/:id/batting-records

## Description

Implement the batch batting-record entry endpoint for admins, including validation and authorization.

## Acceptance Criteria

- [ ] `POST /api/games/:id/batting-records` accepts a batch of records and upserts per game/user
- [ ] Returns 201 on success with created/updated record IDs
- [ ] Validates `pa_result` against allowed enum values
- [ ] Rejects requests from non-admin users with 403
- [ ] Handles duplicate entries gracefully (update instead of insert)

## Technical Details

- Use existing auth middleware to extract user and check admin role
- Validate request body with Zod schema
- Service layer function: `createOrUpdateBattingRecords(gameId, records[])`
- Return `{ recordIds: number[] }`

- Existing auth middleware
