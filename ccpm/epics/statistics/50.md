# Task 003: Backend API — GET /api/stats/me

## Description

Build the authenticated endpoint that returns a player's profile, cumulative batting stats, last-5-game trends, and milestones.

## Acceptance Criteria

- [ ] `GET /api/stats/me` returns profile, cumulative stats, `recent_5_games`, and `milestones`
- [ ] Cumulative stats match PRD fields (`games`, `at_bats`, `hits`, `batting_avg`, `singles`, `doubles`, `triples`, `hr`, `rbi`, `runs`, `walks`, `strikeouts`)
- [ ] `recent_5_games` lists last 5 games with per-game batting average
- [ ] Milestones include at least `hits_to_100` with remaining count
- [ ] Performance < 1 second for players with 50+ games

## Technical Details

- Service layer stat calculator using SQL aggregates
- `batting_avg = hits / at_bats` (exclude walks and sacrifices)
- Use single query or minimal joins to fetch game/user info
- Milestone projections can be simple linear estimates for now

## Dependencies

- Task 001 (batting_records_schema)
- Existing user auth

## Effort Estimate

5 hours

## Definition of Done

- Endpoint returns correct shape per PRD
- Unit/integration tests cover edge cases (zero at-bats, no games)
- Performance verified with seeded data
