# Task 004: Backend API — GET /api/public/leaders

## Description

Implement the public leaderboard endpoint that returns top-N players by a chosen statistical category.

## Acceptance Criteria

- [ ] `GET /api/public/leaders?category=batting_avg&limit=10` returns ranked list
- [ ] Supported categories: `batting_avg`, `hits`, `hr`, `rbi`, `runs`
- [ ] Default `limit=10`, max `limit=50`
- [ ] Query accessible without authentication
- [ ] `rank` starts at 1; ties handled consistently (e.g., standard competition ranking)

## Technical Details

- Reuse stat calculator service from Task 003
- Validate `category` and `limit` with Zod
- SQL aggregate + `ORDER BY` per category
- Minimum threshold: at least 10 at-bats to appear in batting average leaderboard

## Dependencies

- Task 001 (batting_records_schema)
- Sharing service logic with Task 003 helps keep queries consistent

## Effort Estimate

3 hours

## Definition of Done

- Endpoint implemented and tested
- Ranking logic verified
- No auth required
