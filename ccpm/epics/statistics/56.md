# Task 005: Frontend — Admin Batting Record Entry Form

## Description

Build the admin UI for entering batting records per game: a tabular form inside the game detail page allowing batch per-PA input.

## Acceptance Criteria

- [ ] Admin sees a batting entry table on the game detail page
- [ ] Rows include: player select, batting order, position, inning, PA result dropdown
- [ ] Supports adding/removing rows for batch entry
- [ ] Submitting saves all records and shows success/error toast
- [ ] Data reloads after save so admins see latest state

## Technical Details

- React component inside existing game detail route
- Use existing fetch/auth hooks
- Dropdown options: `1B`, `2B`, `3B`, `HR`, `BB`, `K`, `OUT`, `SAC`, `HBP`, `RBI`
- Optimistic clearing of new rows after successful submit

## Dependencies

- Task 002 (batting_records_api)
- Existing game detail page and auth context

## Effort Estimate

5 hours

## Definition of Done

- Component functional and wired to API
- Basic validation (required fields) in place
- Manual smoke test passed
