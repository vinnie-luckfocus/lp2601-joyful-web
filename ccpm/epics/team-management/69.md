# Task 003: Backend API Endpoints

## Description
Expose 3 read-only GET endpoints under `/api/teams/:id` using the new `TeamRepository`, with Zod input validation and consistent API envelope responses.

## Acceptance Criteria
- `GET /api/teams/:id` returns team info, captain, and computed record
- `GET /api/teams/:id/members` returns member list
- `GET /api/teams/:id/games?limit=3` returns recent games (default limit 3, max 10)
- All endpoints validate `:id` as positive integer
- Envelope: `{ success: boolean, data: ..., error: string | null, meta?: object }`

## Technical Details
- Reuse existing Express routing and middleware patterns
- Add rate limiting if not already applied globally
- Return 404 with clear message if team not found

## Dependencies
- Task 002 (repository)

## Effort Estimate
0.5 day

## Definition of Done
- Endpoints reachable locally and return expected JSON shapes
- Integration tests pass for all 3 endpoints
