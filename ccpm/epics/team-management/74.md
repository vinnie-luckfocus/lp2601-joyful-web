# Task 008: Frontend Animations & Lazy Loading

## Description
Add micro-animations to the team page and member grid: logo hover scale, staggered member list fade-in, online status pulse, and lazy-loading for logos/avatars with WebP support where applicable.

## Acceptance Criteria
- Logo hover: scale 1.05 + smooth transition
- Member list items fade in sequentially with 30ms stagger
- Online status dot has continuous pulse/breathe animation
- Images use lazy loading (`loading="lazy"`) with default placeholder
- WebP served when supported (via object storage or `<picture>`)

## Technical Details
- Prefer CSS transitions/keyframes over JS animation libraries
- Lazy load via native `loading="lazy"` on `<img>`
- Keep animations performant (use `transform` and `opacity`)

## Dependencies
- Task 007 (Member Grid)

## Effort Estimate
0.5 day

## Definition of Done
- Animations visible in browser and do not cause layout shifts
- Lighthouse performance score not degraded by new assets
