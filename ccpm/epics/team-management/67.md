# Task 001: Database Schema & Seeds

## Description
Confirm and extend the `teams` table with required fields (`logo_url`, `description`, `division`, `captain_id`), verify `users` table has `team_id`, `jersey_number`, `position`, and `role` fields, and add seed data for at least 2 sample teams with members and games for stats validation.

## Acceptance Criteria
- `teams` table includes: `id`, `name`, `logo_url`, `description`, `division`, `captain_id` (foreign key to `users`)
- `users` table includes: `team_id`, `jersey_number`, `position`, `role` (`captain` | `member`)
- `games` table can be queried by `home_team_id` and `away_team_id`
- Seed script creates 2 teams, each with 5+ members and 3+ completed games

## Technical Details
- Use existing knex/pg migration setup
- Keep schema changes additive (no destructive migrations)
- Seed data should cover win/loss scenarios for record calculations

## Dependencies
- Existing database connection and migration runner

## Effort Estimate
0.5 day

## Definition of Done
- Migrations run successfully locally
- Seed data is queryable and produces predictable win/loss records
