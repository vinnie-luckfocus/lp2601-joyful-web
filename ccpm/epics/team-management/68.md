# Task 002: Backend Repository

## Description
Implement `TeamRepository` with methods to fetch team details, members, and recent games, plus aggregate win/loss/win_rate logic.

## Acceptance Criteria
- `findById(id)` returns team with captain details
- `findMembersByTeamId(id)` returns ordered member list
- `findRecentGamesByTeamId(id, limit)` returns recent completed games
- `getRecordByTeamId(id)` computes `{ wins, losses, win_rate }` from games

## Technical Details
- Follow existing repository pattern in backend
- Keep queries in repository layer; no raw SQL in routes
- Reuse existing DB connection pool

## Dependencies
- Task 001 (schema and seeds)

## Effort Estimate
0.5 day

## Definition of Done
- All repository methods have unit tests (>= 80% coverage)
- Methods run against seeded data and return correct shapes
