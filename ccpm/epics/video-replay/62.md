# Task 62: Backend video list and detail endpoints

## Description

Implement REST endpoints to list videos (with pagination and game info) and retrieve a single video with its highlights.

## Acceptance Criteria

- `GET /api/videos?limit=20&offset=0` returns a list of videos enriched with `game_date`, `home_team`, `away_team`, `highlights_count`
- `GET /api/videos/:id` returns full video details including `highlights: [{ id, time, description }]` sorted by `time` ascending
- Responses use a consistent API envelope (`success`, `data`, `error`, `meta`)
- Invalid IDs return `400 Bad Request`; missing videos return `404 Not Found`
- Unit and integration tests cover both endpoints

## Technical Details

- Add `backend/src/routes/videos.ts` (or equivalent by existing pattern)
- Use Prisma `include: { game: true, highlights: true }` for detail queries
- Use `count` on highlights for list queries to avoid N+1
- Validate `id` with Zod (`uuid` or numeric depending on schema)

## Dependencies

- Task 61 (DB schema and migrations)
- Existing auth middleware for optional user context (if applicable)

6 hours
