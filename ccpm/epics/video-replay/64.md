# Task 64: Backend highlight CRUD endpoints

## Description

Implement endpoints to create and delete highlight markers for a video. Listing is already covered by the detail endpoint in Task 002.

## Acceptance Criteria

- `POST /api/videos/:id/highlights` accepts `{ time, description }` and returns the created highlight
- `time` is validated as a non-negative integer (seconds) and `description` is a non-empty string with max length 200
- `DELETE /api/videos/:id/highlights/:highlightId` removes the highlight
- Both endpoints are admin-only
- Returns `404` if the video or highlight does not exist

## Technical Details

- Validate input with Zod: `time: z.number().int().min(0)` and `description: z.string().min(1).max(200)`
- Reuse the video detail query pattern from Task 002
- Keep endpoints in the same router as videos for consistency

## Dependencies

- Task 61 (DB schema)
- Task 62 (video detail endpoint patterns)

4 hours
