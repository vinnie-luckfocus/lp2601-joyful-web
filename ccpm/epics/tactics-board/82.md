# Task 007: Tactics Board Page and Routing

## Description

Create the `/games/:id/tactics` route and page component. Fetch lineup and tactics data from the backend, compose the `LineupList`, `FieldDiagram`, and `TacticsPanel` components, and handle loading and error states.

## Acceptance Criteria

- [ ] New route `/games/:id/tactics` registered in React Router
- [ ] Page fetches data from `GET /api/games/:id/lineup` on mount
- [ ] Loading state shown while fetching
- [ ] Error state shown on 4xx/5xx with a user-friendly message and retry button
- [ ] On success, page renders lineup list, field diagram, and tactics panel
- [ ] Clicking a position marker in the diagram highlights or scrolls to the corresponding lineup row (optional but preferred)

## Technical Details

- React + TypeScript + Vite
- Use existing API client or fetch wrapper
- Zod validation of API response
- Store data in local component state (or lightweight Zustand slice if pattern exists)

## Dependencies

- Task 003 (backend API)
- Task 004 (FieldDiagram)
- Task 005 (LineupList)
- Task 006 (TacticsPanel)

3-4 hours

- Route and page component committed
- Navigation to `/games/:id/tactics` renders correctly
- Loading and error states verified manually
