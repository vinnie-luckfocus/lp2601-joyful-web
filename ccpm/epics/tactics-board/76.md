# Task 001: Database Schema and Migration for Tactics Board

## Description

Create the database schema and migration for storing game lineups and tactics. Use a dedicated `game_tactics` table with a `game_lineups` child table to store batting order and defensive positions.

## Acceptance Criteria

- [ ] Migration creates `game_tactics` table with columns: `id`, `game_id` (FK to `games`), `general_notes` (text), `signals` (jsonb), `defense_strategy` (text), `created_at`, `updated_at`
- [ ] Migration creates `game_lineups` table with columns: `id`, `game_id` (FK to `games`), `user_id` (FK to `users`), `batting_order` (int 1-9), `position` (varchar), `jersey_number` (varchar), `created_at`, `updated_at`
- [ ] Composite unique constraint on `(game_id, batting_order)`
- [ ] Foreign key constraints with `ON DELETE CASCADE`
- [ ] Migration runs successfully and is reversible

## Technical Details

- PostgreSQL with existing Knex migration setup
- Add indexes on `game_lineups(game_id)` and `game_lineups(user_id)`
- Keep `signals` as JSONB to allow flexible key/value pairs

## Dependencies

- Existing `games` and `users` tables

## Effort Estimate

2-3 hours

## Definition of Done

- Migration file committed
- Migration runs up and down without errors
- Schema verified in local database
