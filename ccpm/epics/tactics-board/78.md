# Task 003: Backend API - GET /api/games/:id/lineup

## Description

Implement the `GET /api/games/:id/lineup` endpoint that returns the complete lineup and tactics for a game. Verify the requesting player belongs to the team associated with the game.

## Acceptance Criteria

- [ ] Endpoint `GET /api/games/:id/lineup` returns 200 with `{ game_id, lineup[], tactics }`
- [ ] `lineup` sorted by `batting_order` and includes `batting_order`, `user_id`, `name`, `position`, `jersey_number`
- [ ] `tactics` includes `general_notes`, `signals`, `defense_strategy`
- [ ] Returns 404 if game does not exist
- [ ] Returns 403 if user does not belong to the game's team
- [ ] Returns 401 if no valid JWT is provided

## Technical Details

- Express + TypeScript backend
- Reuse existing JWT auth middleware
- Reuse existing team-membership authorization logic
- Use Zod for response shape validation in tests
- Log detailed errors on the server, return generic messages to client

## Dependencies

- Task 001 (schema)
- Task 002 (seed data for testing)
- Existing auth and team membership middleware

## Effort Estimate

3-4 hours

## Definition of Done

- Route and controller implemented
- Integration tests pass (80%+ coverage of new code)
- All error cases handled explicitly
