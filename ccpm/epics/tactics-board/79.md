# Task 004: SVG Baseball Field Diagram Component

## Description

Build the `FieldDiagram` SVG component that renders a simplified baseball diamond with 9 defensive position markers. Support responsive scaling, click interactions, and hover states.

## Acceptance Criteria

- [ ] SVG renders a baseball diamond with infield dirt and outfield grass areas
- [ ] 9 position markers placed at approximate real-world coordinates (P, C, 1B, 2B, 3B, SS, LF, CF, RF)
- [ ] Markers are clickable and trigger an `onPositionClick` callback with position data
- [ ] Hover state changes marker color from white/red border to red fill
- [ ] Component accepts a `lineup` prop to display player info per position
- [ ] Responsive: SVG scales to parent container width while preserving aspect ratio

## Technical Details

- React + TypeScript
- SVG-only (no Canvas or external diagram libraries)
- Tailwind CSS for styling where possible; inline SVG styles for precise positioning
- MLB color palette from PRD: grass `#2D8659`, dirt `#8B4513`, marker `#FFFFFF` border `#BF0D3E`, hover `#BF0D3E`

## Dependencies

- None (pure UI component)

## Effort Estimate

4-5 hours

## Definition of Done

- Component file created under frontend source
- Unit tests for click and hover interactions pass
- Visual inspection in Storybook or dev server confirms layout
