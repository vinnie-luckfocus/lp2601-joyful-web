# Task: Create games and game_attendance tables with seed data

## Description

Create the database schema for the game schedule system, including the `games` table and the `game_attendance` junction table. Add seed data for local development and write migration rollback tests.

## Acceptance Criteria

- [ ] `games` table exists with columns: `id` (PK), `scheduled_at` (timestamptz), `location` (text), `home_team_id` (FK), `away_team_id` (FK), `status` (enum: scheduled | live | completed), `created_at`, `updated_at`
- [ ] `game_attendance` table exists with columns: `id` (PK), `game_id` (FK), `user_id` (FK), `status` (confirmed | declined), `created_at`, `updated_at`
- [ ] Unique index on `game_attendance(game_id, user_id)` to enforce one status per user per game
- [ ] Index on `games.scheduled_at` for fast chronological queries
- [ ] Index on `game_attendance(game_id, status)` to support attendee count aggregation
- [ ] Seed data includes at least 10 upcoming games and realistic attendance records
- [ ] Seed data references valid `home_team_id` and `away_team_id` (create team seed rows if the `teams` table does not already exist)
- [ ] Document the `pending` attendance computation strategy (team roster table vs all authenticated users) and seed any required roster data
- [ ] Migration rollback test verifies `DOWN` migration removes both tables cleanly

## Technical Details

- Use existing migration conventions in `database/migrations/`
- Follow existing seed patterns in `database/seeds/`
- Add rollback test in `tests/backend/database/schema.test.ts` or equivalent
- If `teams` table is missing from prior epics, include it in this migration or seed set
