# Task: Write E2E tests for critical schedule flows

## Description

Write end-to-end tests covering the complete game schedule user journey: viewing the schedule, filtering by month, opening a game, signing up, and verifying the attendee count updates.

## Acceptance Criteria

- [ ] E2E test: unauthenticated user can view the public schedule
- [ ] E2E test: authenticated user can filter schedule by month and see correct games
- [ ] E2E test: empty month view shows appropriate empty state (no games message)
- [ ] E2E test: user clicks a game to open detail page
- [ ] E2E test: user clicks "参加", sees optimistic UI update, and attendee count increments
- [ ] E2E test: user clicks "不参加", status updates, and attendee count decrements
- [ ] E2E test: signup is blocked within 2 hours of game start with appropriate message
- [ ] E2E test: optimistic UI rolls back gracefully when the attend API returns a network/server error
- [ ] Tests run reliably against local dev stack

## Technical Details

- Use existing E2E framework (Playwright or project default)
- Seed database with predictable test data before each test
- Use existing test users/credentials from auth setup
- Capture screenshots/traces on failure for debugging

## Dependencies

- [ ] 005 — routes must be live
- [ ] 006 — GameSchedulePage must exist
- [ ] 007 — GameDetailPage and attendance buttons must exist
- [ ] 008 — ScheduleGameCard must exist
