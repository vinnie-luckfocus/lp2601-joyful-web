# Task: Build GameDetailPage with attendance sections and animations

## Description

Build the single-match detail page showing full game info, attendance lists split by status, and action buttons with MLB-style feedback animations.

## Acceptance Criteria

- [ ] `GameDetailPage` displays game time, location, home/away teams, and game status badge
- [ ] Card background uses `#FFFFFF` per PRD MLB style
- [ ] Attendees are shown in three sections: "已确认" (green), "未回复" (gray), "不参加" (optional muted style)
- [ ] Each section shows avatar + name with a horizontal row and `+N` overflow when > 5 avatars
- [ ] `AttendanceButtons` show "参加" and "不参加" with `active:scale-95` click feedback and `#BF0D3E` → `#A00B34` hover transition
- [ ] Successful signup triggers a green checkmark pop animation + subtle shake on the button/row
- [ ] Optimistic UI updates the local attendee list immediately after button click
- [ ] API errors revert the optimistic update and show a user-friendly toast/alert

## Technical Details

- Avatars fall back to user initials if `avatar_url` is missing (backend must return `name` for initials generation)
- Use optimistic UI pattern via `useAttendance` hook
- Game status badges use PRD colors: scheduled = `#041E42`, live = `#2D8659`, completed = `#A0AEC0`
- Page is reachable at `/games/:id`

## Dependencies

- [ ] 004 — attendance API must exist
- [ ] 005 — routes must be registered
- [ ] 008 — `GameCard` sub-components and `ScheduleGameCard` must be available
