## Technical Details

### 核心表结构
```sql
-- 用户表
users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100),
  phone VARCHAR(20),
  avatar_url VARCHAR(255),
  team_id INTEGER REFERENCES teams(id),
  jersey_number INTEGER,
  position VARCHAR(50),
  role VARCHAR(20) DEFAULT 'player', -- admin, player
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- 球队表
teams (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  logo_url VARCHAR(255),
  description TEXT,
  division VARCHAR(50), -- 大联盟/小联盟
  wins INTEGER DEFAULT 0,
  losses INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- 比赛表
games (
  id SERIAL PRIMARY KEY,
  scheduled_at TIMESTAMP NOT NULL,
  location VARCHAR(100) NOT NULL,
  home_team_id INTEGER REFERENCES teams(id),
  away_team_id INTEGER REFERENCES teams(id),
  home_score INTEGER,
  away_score INTEGER,
  status VARCHAR(20) DEFAULT 'scheduled', -- scheduled, ongoing, completed, cancelled
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)
```

### 实现方法
1. 使用 dbdiagram.io 或类似工具绘制 ER 图
2. 编写完整的 SQL DDL 文件
3. 导出 PNG/SVG 格式的 ER 图
4. 将 SQL 文件保存到 database/migrations/
