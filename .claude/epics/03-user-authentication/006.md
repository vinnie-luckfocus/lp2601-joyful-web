# Task: Implement MLB-style login modal and first-login password change

## Description
Replace or augment the current `/login` full-page login with a modal-based login experience on the homepage. Apply MLB-style red accents (`#BF0D3E`), focus transitions, and a shake-on-error animation. Add an inline or modal first-login password change form that appears when `is_first_login` is `true`.

## Acceptance Criteria
- [ ] Login modal overlay uses `rgba(4, 30, 66, 0.7)` background
- [ ] Login button uses `#BF0D3E` with hover `#A00B34`
- [ ] Input focus border transitions to `#BF0D3E`
- [ ] Modal has slide-in + fade animation (200ms)
- [ ] Error state triggers a shake animation with friendly error message text
- [ ] Logo height is 48px and centered
- [ ] Successful login redirects the user to a personal-related page (e.g., team or player dashboard) per PRD Story 1
- [ ] First-login users see a password change form before accessing protected content
- [ ] After successful password change, user is logged out and modal shows the login form again (requires re-login per PRD Story 2)
- [ ] `/login` route remains as a deep-link fallback or redirects appropriately

## Technical Details
- Files: `frontend/src/pages/Login/index.tsx`, `frontend/src/pages/Home/index.tsx`, and potentially new `frontend/src/components/LoginModal/`
- Use Tailwind CSS for styling and Framer Motion (or CSS transitions) for animations
- Integrate with `useAuthStore` for login and `changePassword` actions
- Ensure accessibility (focus trap, ESC to close, ARIA labels)
