# Task: Implement POST /api/auth/change-password

## Description
Create a new authenticated endpoint `POST /api/auth/change-password` that allows users to change their password. On success, set `is_first_login` to `false` so the forced password change flow is completed.

## Acceptance Criteria
- [ ] New endpoint `POST /api/auth/change-password` exists and is protected by auth middleware
- [ ] Request body is validated with Zod (`old_password`, `new_password`)
- [ ] `new_password` Zod validation enforces minimum length of 6 characters and maximum of 72 bytes (bcrypt truncation limit)
- [ ] `old_password` is verified against the stored bcrypt hash
- [ ] `new_password` is bcrypt-hashed before storage
- [ ] On success, `is_first_login` is set to `false` in the database
- [ ] API response time for `POST /api/auth/change-password` is < 500ms under normal load
- [ ] Appropriate error responses for invalid current password or validation failures

## Technical Details
- File: `backend/src/routes/auth.ts`
- Use existing `bcrypt` utilities in `backend/src/utils/password.ts`
- Apply the same rate-limiting strategy as login if applicable
- New password validation: minimum 6 characters, maximum 72 bytes (bcrypt limit)
- Align API field names with PRD spec: `old_password`, `new_password`
