# Task: Fix ProtectedRoute loading flash and axios 401 redirect loop

## Description
Fix the two known auth boundary UX bugs: `ProtectedRoute` redirects before `checkAuth()` resolves (causing a flash of login), and the axios 401 interceptor unconditionally redirects to `/login` which can cause a hard reload loop.

## Acceptance Criteria
- [ ] `ProtectedRoute` waits for auth initialization (`isLoading` / `hydrated` state) before deciding to redirect
- [ ] During initialization, a loading spinner or null is rendered instead of the login redirect
- [ ] Axios 401 interceptor only redirects to `/login` if the current pathname is not already `/login`
- [ ] No infinite reload loops occur when a 401 fires on the login page

## Technical Details
- Files: `frontend/src/components/ProtectedRoute/index.tsx`, `frontend/src/utils/axios.ts`
- Add an `isLoading` or `initialized` flag to `useAuthStore` or `ProtectedRoute`
- In axios interceptor, check `window.location.pathname !== '/login'` before setting `window.location.href`
- Consider using `useNavigate` from React Router instead of `window.location.href` if possible
