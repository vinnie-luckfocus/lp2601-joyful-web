# Task: Implement attendance API with cutoff and concurrency safety

## Description

Build the attendance endpoints with Zod validation, enforce the 2-hour signup deadline, protect against race conditions with atomic transactions, and add rate limiting on the signup endpoint.

## Acceptance Criteria

- [ ] Write integration tests **first** (RED), then implement endpoints (GREEN), then refactor. Tests must cover: successful signup, cutoff rejection, concurrent signups (simulate 10 parallel requests), invalid status, invalid `gameId`, and rate limiting
- [ ] `POST /api/games/:id/attend` requires JWT auth and validates body `{ status: 'confirmed' | 'declined' }` via Zod
- [ ] Validates `gameId` path parameter format before database access
- [ ] Rejects requests if current time is within 2 hours of `games.scheduled_at` with a user-friendly, safe error message (no internal details leaked)
- [ ] Atomic transaction wraps cutoff check and `INSERT ... ON CONFLICT (game_id, user_id) DO UPDATE`; uses `SELECT ... FOR UPDATE` on the `games` row to prevent cutoff race conditions
- [ ] Rate limiter allows max 10 requests per minute per authenticated `userId` (extracted from JWT) on this endpoint
- [ ] `GET /api/games/:id/attendance` requires JWT auth and returns `{ confirmed: [...], declined: [...], pending: [...] }`
- [ ] `pending` is computed based on the team roster defined in Task 001; if no roster exists, fallback to all authenticated users minus confirmed/declined
- [ ] `declined` list is visible to all authenticated users (document this decision) or restrict as decided in the epic

## Technical Details

- Use `pg.Pool` transactions: `BEGIN; SELECT scheduled_at FROM games WHERE id = $1 FOR UPDATE; ... INSERT/UPDATE ... COMMIT;`
- Apply `express-rate-limit` or existing project rate-limiting middleware keyed by JWT `userId`
- Return standard API envelope on all responses
- Error messages must not leak internal details (stack traces, SQL, etc.)
- CSRF is implicitly mitigated by requiring the `Authorization: Bearer` header on all state-changing requests
