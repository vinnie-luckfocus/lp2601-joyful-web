# Task: Extend public API for full schedule

## Description

Extend `backend/src/routes/public.ts` and `backend/src/services/publicApi.ts` to serve the full season game schedule publicly, returning all games (not limited to 10) with standard response envelope and cache headers.

## Acceptance Criteria

- [ ] Write integration tests **first** (RED), then implement endpoint (GREEN), then refactor. Tests must cover: response shape, sorting, cache header, and optional `limit` behavior
- [ ] `GET /api/public/games` returns all season games sorted by `scheduled_at` ascending by default
- [ ] Supports optional `?limit={n}` query parameter to preserve homepage usage (default: no limit)
- [ ] Response envelope follows `{ success, data, error, meta }` pattern with `meta.total_count` included
- [ ] Each game includes: `id`, `scheduled_at`, `location`, `home_team`, `away_team`, `status`
- [ ] Endpoint returns `Cache-Control: public, max-age=60` header
- [ ] Query uses parameterized SQL and leverages the `games.scheduled_at` index

## Technical Details

- Extend existing `publicApi.ts` service rather than creating a new service file
- Reuse existing route and test conventions from `backend/src/routes/public.ts`
- No authentication required
