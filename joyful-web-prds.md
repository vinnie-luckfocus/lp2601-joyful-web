| :--- | :--- | :--- |

## 1. 产品概述

### 1.1 产品定位
Joyful 棒球俱乐部联赛网站是专为成人业余棒球联赛打造的**赛前准备与赛后复盘平台**。

**核心价值**：让队员知道什么时候打、跟谁打、怎么打，打完能看录像和数据分析。

### 1.2 目标用户

| 用户角色 | 使用场景 | 核心需求 |
| :--- | :--- | :--- |
| **队员** | 赛前查看赛程战术，赛后复盘数据 | 赛程、战术、报名、录像、个人数据 |
| **队长/管理员** | 管理球队信息，录入比赛数据 | 球队管理、数据录入、视频上传 |

### 1.3 实现阶段

| 阶段 | 内容 |
| :--- | :--- |
| **第一阶段** | 管理员录入球队、队员、赛程数据，为队员分配账号 |
| **第二阶段** | 开放会员注册系统，新用户可自助注册并申请入队 |

## 2. 功能需求

### 2.1 联赛赛程

**功能描述**：展示所有比赛信息，支持队员查看和报名。

**核心功能**：
- 以日历/列表形式展示全部赛程
- 单场比赛信息：时间、场地、对阵双方、当前报名人数
- 队员可点击"参加"或"不参加"报名
- 显示已报名队员名单

### 2.2 球队展示

**功能描述**：每个球队拥有独立的展示页面。

**球队信息**：
- **队名**：球队名称（如：Joyful A队）
- **队徽**：球队Logo图片
- **球队简介**：一段介绍文字，可包括球队历史、风格、口号等
- **联盟级别**：所属联盟（大联盟/小联盟）
- **赛季战绩**：胜场/负场/胜率

**队员视角**：
- 查看自己所属球队的信息
- 查看球队成员列表
- 查看下场比赛的战术安排和个人位置

### 2.3 战术板

**功能描述**：展示球队战术安排。

**核心功能**：
- **位置安排**：显示每位队员的守备位置
- **棒次安排**：显示打击顺序
- **战术说明**：文字形式的战术要点（如：对左投策略、双杀战术暗号等）
- **布阵图**：可视化球场位置图

### 2.4 视频回放

**功能描述**：上传和观看比赛录像。

**核心功能**：
- 比赛录像上传（支持MP4格式）
- 视频播放（基础播放控制：播放、暂停、进度条、倍速）
- 精彩时刻标记（管理员可标记时间点并添加描述，如"03:15 张三本垒打"）
- 按比赛日期浏览历史录像

### 2.5 数据记录与统计

**功能描述**：记录比赛数据并生成统计。

**数据录入（管理员/队长）**：
- 简化版打席记录：棒次、姓名、位置、打席结果
- 打席结果选项：一垒安打、二垒安打、三垒安打、本垒打、三振、四坏球、飞球出局、滚地出局、双杀、牺牲打

**个人数据展示（队员）**：
- 累计数据：出赛场次、打席数、安打数、打击率、打点、得分
- 详细数据：一垒/二垒/三垒/本垒打分项统计
- 最近趋势：近5场打击率变化曲线

## 3. UI/UX 设计规范

### 3.1 整体风格

**设计原则**：
- **简洁现代**：界面干净，留白充足，信息层次清晰
- **专业运动感**：体现棒球运动的活力与团队精神
- **高质感视觉**：精致的图标、圆角设计、柔和阴影

**色彩方案**：
- **主色调**：深蓝色（#1a365d）—— 代表专业、稳重
- **辅色调**：活力橙（#ed8936）—— 代表运动、激情，用于强调和CTA按钮
- **背景色**：浅灰白（#f7fafc）—— 干净、不刺眼
- **文字色**：深灰（#2d3748）—— 易读性优先
- **强调色**：成功绿（#38a169）、警告黄（#ecc94b）、错误红（#e53e3e）

### 3.2 动效设计

**页面转场**：
- 页面切换使用淡入淡出（fade-in/out），时长200-300ms
- 列表加载使用依次滑入（stagger slide-up），营造流畅感

**交互动效**：
- 按钮悬停：轻微上浮（translateY -2px）+ 阴影加深
- 卡片悬停：微妙放大（scale 1.02）+ 阴影增强
- 点击反馈：按钮按下时缩放（scale 0.98）
- 数据更新：数字变化使用计数动画

**数据可视化动效**：
- 图表加载时从0动画增长到实际值
- 雷达图逐项绘制，增强仪式感
- 趋势图线条从左到右绘制

**加载状态**：
- 骨架屏（Skeleton Screen）替代传统loading
- 图片加载使用模糊渐显（blur-up lazy loading）

### 3.3 组件规范

**卡片设计**：
- 圆角：12-16px
- 阴影：0 4px 6px rgba(0,0,0,0.05)
- 悬停阴影：0 10px 15px rgba(0,0,0,0.1)
- 背景：纯白或轻微渐变

**按钮设计**：
- 主按钮：填充色（橙色）+ 白色文字 + 圆角（8px）
- 次按钮：描边风格 + 主色文字
- 幽灵按钮：仅文字 + 悬停下划线
- 尺寸：大按钮（48px高）、中按钮（40px高）、小按钮（32px高）

**表单设计**：
- 输入框：底部边框或圆角矩形，聚焦时主色边框
- 标签：浮动标签（Floating Label）或顶部固定标签
- 错误提示：红色文字 + 图标，实时验证

**导航设计**：
- 底部Tab导航（移动端）或顶部导航栏（Web端）
- 当前页面图标高亮（主色）
- 徽标（Badge）提示未读消息或待报名比赛

## 4. 页面设计详述

### 4.1 登录页

**布局**：
- 中央卡片式布局，最大宽度400px
- 顶部：Joyful Logo + 标语
- 中部：账号密码输入框 + 登录按钮
- 底部：忘记密码链接（第一阶段隐藏）

**动效**：
- 页面加载时卡片从下方滑入
- 输入框聚焦时边框颜色平滑过渡

### 4.2 赛程页（首页）

**布局**：
- 顶部：月份切换器
- 主体：日历网格或时间线列表
- 比赛卡片：日期、时间、对阵、场地、报名人数

**视觉**：
- 不同状态用颜色区分：未开始（蓝色）、进行中（绿色）、已结束（灰色）
- 我参加的比赛卡片高亮显示

**动效**：
- 切换月份时比赛卡片交错滑入
- 点击报名按钮时显示成功动画（勾选图标+轻微震动）

### 4.3 球队页

**布局**：
- 顶部：队徽（大尺寸）+ 队名 + 赛季战绩
- 中部：球队简介（可展开/收起）
- 下部：球队成员网格头像 + 下场比赛战术入口

**视觉**：
- 队徽区域使用球队主色作为背景渐变
- 成员头像使用统一尺寸的圆形，在线状态用绿点标识

**动效**：
- 队徽悬停时轻微放大并显示球队简介预览
- 成员列表加载时依次淡入

### 4.4 战术板页

**布局**：
- 左侧：棒次列表（1-9棒 + 姓名 + 位置）
- 右侧/下方：球场布阵图（可视化位置分布）
- 底部：战术说明文字区

**视觉**：
- 棒次列表突出显示"我"的位置（高亮背景）
- 布阵图使用简化棒球场示意图，位置点可点击查看队员

**动效**：
- 布阵图加载时位置点依次弹出
- 点击位置点时显示队员卡片（滑入动画）

### 4.5 视频页

**布局**：
- 顶部：最近比赛视频大卡片（封面图+播放按钮）
- 下部：历史视频列表（缩略图+日期+比分）
- 播放页：全宽视频播放器 + 下方精彩时刻时间轴

**视觉**：
- 视频封面使用比赛精彩瞬间截图
- 播放按钮使用圆形主色按钮 + 三角播放图标
- 时间轴用垂直线连接各个标记点

**动效**：
- 点击播放按钮时按钮扩散消失，视频渐入
- 精彩时刻标记点悬停时显示预览缩略图

### 4.6 数据页

**布局**：
- 顶部：个人头像 + 姓名 + 背号
- 数据卡片区：累计数据网格（6-8个关键指标）
- 图表区：打击率趋势折线图 + 五维能力雷达图

**视觉**：
- 数据卡片使用大字体数字 + 小字体标签
- 趋势图使用渐变色填充区域
- 雷达图使用半透明填充 + 发光描边

**动效**：
- 页面进入时数字从0计数动画到实际值
- 图表线条/区域绘制动画
- 雷达图顶点依次连接动画

### 4.7 管理后台

**布局**：
- 侧边导航栏（固定）+ 右侧内容区
- 数据录入表格（简洁行设计）
- 视频上传区（拖拽上传 + 进度条）

**视觉**：
- 后台使用更紧凑的布局
- 表格行悬停高亮
- 操作按钮（编辑/删除）在行尾，默认灰色，悬停显色

## 5. 设备适配规范

### 5.1 响应式断点

| 设备类型 | 断点范围 | 布局策略 |
| :--- | :--- | :--- |
| **手机** | < 640px | 单列布局，底部Tab导航，卡片全宽 |
| **平板** | 640px - 1024px | 双列布局，侧边导航或顶部导航 |
| **桌面** | > 1024px | 多列布局，固定侧边栏，最大内容区宽度1200px |

### 5.2 移动端优先设计

**触摸优化**：
- 所有可点击元素最小尺寸 44x44px
- 按钮间距至少 8px，防止误触
- 滑动操作支持（如：左滑报名/取消）

**移动端特有设计**：
- 底部固定导航栏（4-5个主要入口）
- 顶部固定标题栏（显示当前页面名）
- 下拉刷新手势支持
- 上拉加载更多

**输入优化**：
- 数字输入使用数字键盘
- 日期选择使用原生日期选择器
- 减少手动输入，多用选择器

### 5.3 桌面端增强

**大屏优化**：
- 赛程页使用日历网格视图（手机用列表）
- 数据页并排显示多个图表
- 战术板左右分栏显示（手机上下堆叠）

**键盘快捷键**（可选）：
- `ESC`：关闭弹窗
- `←/→`：切换比赛/视频

### 5.4 跨设备一致性

**状态同步**：
- 同一账号在不同设备登录，数据实时同步
- 报名操作在任一设备完成，其他设备立即更新

**性能要求**：
- 首屏加载时间 < 2秒（3G网络）
- 图片懒加载，优先加载可视区域内容
- 视频按需加载，不预加载大文件

## 6. 技术架构建议

### 6.1 前端技术栈

| 层级 | 技术选择 | 说明 |
| :--- | :--- | :--- |
| 框架 | React 18 + TypeScript | 类型安全，生态成熟 |
| 样式 | Tailwind CSS | 原子化CSS，快速开发 |
| 动效 | Framer Motion | React动画库，声明式API |
| 图表 | Recharts | React图表库，支持动画 |
| 状态管理 | Zustand 或 React Query | 轻量级状态管理 |
| 构建工具 | Vite | 快速热更新，优化构建 |

### 6.2 后端技术栈

| 层级 | 技术选择 | 说明 |
| :--- | :--- | :--- |
| 框架 | Node.js + Express 或 Python + FastAPI | RESTful API |
| 数据库 | PostgreSQL | 关系型数据，支持JSON字段 |
| 视频存储 | 阿里云OSS / 腾讯云COS | 对象存储，CDN加速 |
| 认证 | JWT | 无状态认证，支持扩展 |

### 6.3 数据库预留（支持后续会员系统）

核心预留字段：
- `users.is_manual_created`：区分管理员录入/自主注册
- `users.email / phone`：第二阶段启用
- `users.status`：pending/active/inactive

## 7. 开发计划

### Phase 0：数据初始化（1周）
- 设计数据库表结构
- 开发管理后台基础功能
- 录入球队、队员、赛程数据
- 为队员生成初始账号

### Phase 1：核心功能（5-6周）
- 用户登录（分配账号）
- 赛程展示与报名
- 球队信息展示
- 战术板展示
- 比赛数据录入
- 基础数据统计

### Phase 2：体验完善（2-3周）
- 视频上传与播放
- 精彩时刻标记
- 数据可视化图表
- 响应式适配优化
- UI动效实现

### Phase 3：会员系统（3-4周，第二阶段）
- 开放用户注册
- 邮箱/手机号验证
- 历史数据绑定
- 入队申请与审核
