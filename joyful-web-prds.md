## 1. 项目概述

### 1.1 项目背景
本俱乐部拥有 2 个标准棒球场和 1 个标准垒球场，业务覆盖青少年青训与成人业余联赛。目前面临的主要痛点包括：
1. **管理松散**：训练缺乏系统性，出勤率低，缺乏数据沉淀。
2. **组队复杂**：新人入队机制依赖人工协调，效率低。
3. **体验断层**：比赛与训练脱节，缺乏数字化手段提升用户粘性。
4. **资源闲置**：场地与教练资源未通过数字化手段最大化利用。

### 1.2 产品目标
1. **运营自动化**：实现会员自动分组、赛程自动生成、考勤自动统计。
2. **赛训专业化**：建立“以赛代练、以练促赛”的闭环，通过数据驱动训练计划。
3. **用户社群化**：通过邀请激励、家庭账户、球队社交增强用户归属感。
4. **商业可持续**：通过精细化会员管理和增值服务（私教、装备、视频分析）提升营收。

### 1.3 适用范围
*   **前端**：iOS/Android 移动端 App（会员/教练/裁判端）。
*   **后端**：Web 管理后台（运营/管理员端）。

| 角色 | 描述 | 核心诉求 |
| :--- | :--- | :--- |

## 3. 核心业务逻辑架构

### 3.1 会员生命周期与自动组队机制

#### 3.1.1 会员类型定义
*   **Student**: 仅限青训课程，无联赛资格。
*   **Individual**: 独立成人会员，可加入联赛。
*   **Family_Main**: 家庭户主，可绑定 N 个子账号（Student 或 Individual）。
*   **Rookie (新人)**: 注册时间 < 365天 且 `Team_ID == NULL`。

#### 3.1.2 新人流转状态机
1.  **进入新人池**：新用户注册缴费后，自动标记为 `Rookie`，加入虚拟队伍“新人训练营”。
2.  **自动成队触发**：
    *   系统每日扫描 `Rookie` 池中状态为 `Active` 的人数。
    *   当人数 $\ge$ 9 人时，触发`Create_New_Team`事件。
    *   系统生成新队伍 ID，初始等级设为 `Minor League (小联盟)`。
    *   通知管理员审核并分配教练。
3.  **租借机制 (Loan System)**：
    *   **场景**：正式队伍（大/小联盟）出现空缺（<9人上场阵容）。
    *   **流程**：
        1. 队长在 App 发起“租借请求”，设定位置需求（如：急需投手）。
        2. 系统从 `Rookie` 池中筛选符合位置偏好且本周空闲的新人。
        3. 向新人推送“试打邀请”。
        4. 新人接受 -> 临时绑定至该场比赛阵容。
        5. 赛后数据计入个人生涯，但不计入球队赛季积分；租借结束自动解绑。

### 3.2 赛季与赛制引擎

*   **赛季周期**：每年 3月 - 11月。
*   **第一阶段：双循环积分赛 (Regular Season)**
    *   **规则**：每两支队伍主客场各赛一场。
    *   **积分**：胜=3, 平=1, 负=0。
    *   **排名依据**：积分 > 净胜分 > 相互战绩。
*   **第二阶段：分组淘汰赛 (Playoffs)**
    *   **触发**：常规赛结束后。
    *   **分组**：前 4 名进入“争冠组”，后 4 名进入“排位组”（具体名额可配）。
    *   **赛制**：3局2胜制 (Best of 3)。
    *   **对阵生成**：系统根据种子排名自动生成 Bracket（1vs4, 2vs3...）。

### 3.3 赛训一体化闭环 (Match-Training Loop)

为解决“训练松散”问题，实施以下强制逻辑：
1.  **周六比赛日**：暴露技术短板 -> 数据录入 App。
2.  **周日训练日**：
    *   **课前**：App 推送基于上周比赛数据的“针对性预习视频”。
    *   **课中**：教练执行结构化教案，App 记录考勤与表现。
    *   **课后**：生成个人改进报告。
3.  **出勤挂钩**：
    *   月度出勤率 < 70%：限制下周比赛首发报名资格。
    *   连续 2 次无故缺席：暂停会员资格 1 周。

### 3.4 邀请激励体系 (Referral Program)

*   **规则**：老会员邀请新人成功注册并缴费，老会员获得奖励。
*   **奖励形式**：
    *   年费顺延（如 +1个月）。
    *   或 等额积分/代金券（可用于购买装备或私教课）。
*   **风控**：
    *   同一设备/IP 限制注册数量。
    *   家庭子账号不计入邀请奖励。
    *   奖励在新人度过“冷静期”（如7天）后发放。

## 4. 功能模块详解 (App 端)

### 4.1 首页 (Home)
*   **顶部栏**：天气信息（影响户外比赛）、公告跑马灯。
*   **今日日程卡片**：
    *   显示今日训练/比赛时间、地点、对手。
    *   操作按钮：“签到”、“请假”、“查看详情”。
*   **快捷入口**：我的球队、赛程表、训练打卡、会员中心、邀请好友。
*   **动态流**：俱乐部新闻、精彩集锦、排行榜更新。

### 4.2 赛事中心 (League)
*   **赛程日历**：支持按月/周/列表视图，筛选联盟（Major/Minor/Rookie）。
*   **比赛详情页**：
    *   **基本信息**：时间、场地（一键导航）、裁判、气象。
    *   **实时数据看板**：
        *   比分、局数、出局数、垒包情况。
        *   球员数据：AB, H, HR, RBI, SO, BB 等。
    *   **视频回放**：
        *   Phase 1: 嵌入 HLS 播放器，播放管理员上传的录像。
        *   Phase 2: 预留直播接口，显示“Live”标签。
    *   **历史交锋**：两队过往战绩对比。
*   **积分榜**：团队排名、个人数据排行榜（打击率、防御率等）。

### 4.3 训练中心 (Academy)
*   **课程表**：
    *   区分：青训梯队、成人基础班、高阶战术班。
    *   状态：已满、可报名、候补。
*   **我的训练计划**：
    *   **本周重点**：教练发布的视频教案（如：滑步技巧）。
    *   **课前测验**：3-5道选择题，确保理解战术意图。
    *   **课后反馈**：教练评分与评语。
*   **场地预订**：
    *   可视化展示 3 块场地的空闲时段。
    *   权限控制：仅高等级会员或教练可预订非训练时段。

### 4.4 球队空间 (Team Hub)
*   **成员列表**：显示队员头像、位置、近期状态（红/绿灯表示出勤率）。
*   **战术板 (Playbook)**：
    *   图文/视频形式的暗号表、防守布阵图。
    *   支持离线下载。
*   **更衣室 (Chat)**：
    *   仅限本队成员的群组聊天。
    *   功能：投票（聚餐/队服）、活动报名、文件共享。
*   **荣誉墙**：本月 MVP、最佳进步奖、全勤奖展示。

### 4.5 个人中心 (Profile)
*   **数字会员卡**：二维码（入场核销）、会员类型、有效期。
*   **家庭管理**：主账号可切换查看子账号（子女）的数据和课程。
*   **生涯数据**：
    *   累计出场、总安打、本垒打、盗垒。
    *   雷达图：打击、守备、跑垒、力量、速度五维能力值。
*   **邀请好友**：生成专属海报/链接，查看邀请进度与奖励余额。
*   **钱包/订单**：会费缴纳记录、积分明细、商城订单。

## 5. 功能模块详解 (Web 管理后台)

### 5.1 会员管理
*   **用户列表**：搜索、筛选（类型、状态、所属队伍）。
*   **家庭关系绑定**：手动或自动关联主/子账号。
*   **新人审核**：监控新人池人数，确认自动成队或手动干预。

### 5.2 球队与教练管理
*   **队伍配置**：创建队伍、设定联盟级别、绑定教练。
*   **教练考核**：查看教练发布的教案数量、队员评分、队伍胜率。
*   **租借管理**：审批跨队租借请求，查看租借历史。

### 5.3 赛事引擎
*   **赛程生成器**：
    *   输入：参赛队伍、场地可用时间、避讳日期。
    *   输出：双循环赛程表，支持手动拖拽调整。
*   **季后赛配置**：设置晋级规则、分组阈值。
*   **数据修正**：管理员拥有最高权限，可修正现场录入的错误数据。

### 5.4 内容与视频管理 (CMS)
*   **视频上传**：支持断点续传，自动转码为 HLS 格式。
*   **标签管理**：为视频打标（如：#全垒打 #精彩守备），便于检索。
*   **公告发布**：全站推送或指定人群推送。

### 5.5 财务报表
*   **收入统计**：会费、单次训练费、装备销售、赞助收入。
*   **支出统计**：教练薪资、场地维护、营销费用。
*   **邀请奖励结算**：每月自动生成待发放奖励清单。

## 6. 非功能性需求

### 6.1 性能要求
*   **并发支持**：支持至少 500 人同时在线，周末比赛高峰期视频流流畅无卡顿。
*   **响应时间**：App 页面加载时间 < 1.5秒，数据查询 < 0.5秒。

### 6.2 安全性
*   **数据隐私**：未成年人数据加密存储，默认不对外公开详细个人信息。
*   **权限控制**：基于 RBAC (Role-Based Access Control) 的权限模型，严格隔离不同角色数据。
*   **防刷机制**：API 接口增加频率限制，防止恶意刷取邀请奖励。

### 6.3 兼容性
*   **移动端**：iOS 14+, Android 10+。
*   **Web 端**：Chrome, Safari, Edge 最新两个版本。

## 7. 数据埋点与统计指标 (KPIs)

| 指标类别 | 关键指标 | 目标值 |
| :--- | :--- | :--- |
| **用户增长** | 月活跃用户数 (MAU) | 季度环比增长 10% |
| | 新人转化率 | 注册到付费转化率 > 30% |
| | 邀请成功率 | 每个活跃用户平均邀请 0.5 人 |
| **参与度** | 训练出勤率 | > 85% |
| | 人均周打开次数 | > 5 次 |
| | 视频观看时长 | 人均每周 > 10 分钟 |
| **商业化** | 续费率 | 年费续费率 > 70% |
| | 增值服务渗透率 | 私教/装备购买率 > 15% |

## 8. 开发路线图 (Roadmap)

### Phase 1: 基础建设与规范化 (MVP) - 2.5 个月
*   **核心功能**：会员注册/登录、家庭账户、基础赛程展示、手动比分录入、简单签到。
*   **重点突破**：新人自动组队逻辑后端实现、邀请激励基础框架。
*   **交付物**：App v1.0, Web Admin v1.0。

### Phase 2: 赛训深化与数据驱动 - 2 个月
*   **核心功能**：结构化训练教案、课前/课后任务、考勤与出场资格挂钩、个人数据雷达图、球队空间（聊天/战术板）。
*   **重点突破**：解决“训练松散”问题，强制推行数字化考勤与预习。
*   **交付物**：App v2.0。

### Phase 3: 智能化与体验升级 - 3 个月
*   **核心功能**：视频直播/回放优化、AI 动作分析接口预留、智能排阵建议、装备商城、赞助商广告位。
*   **重点突破**：提升用户体验，探索新的盈利模式。
*   **交付物**：App v3.0。
