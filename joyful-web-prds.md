## 1. 产品概述

### 1.1 产品定位
Joyful 棒球俱乐部联赛网站是专为成人业余棒球联赛打造的**赛前准备与赛后复盘平台**。

**核心价值**：让队员知道什么时候打、跟谁打、怎么打，打完能看录像和数据分析。

### 1.2 目标用户

| 用户角色 | 使用场景 | 核心需求 |
| :--- | :--- | :--- |
| **队员** | 赛前查看赛程战术，赛后复盘数据 | 赛程、战术、报名、录像、个人数据 |
| **队长/管理员** | 管理球队信息，录入比赛数据 | 球队管理、数据录入、视频上传 |

### 1.3 实现阶段

| 阶段 | 内容 |
| :--- | :--- |
| **第一阶段** | 管理员录入球队、队员、赛程数据，为队员分配账号 |
| **第二阶段** | 开放会员注册系统，新用户可自助注册并申请入队 |

## 2. 功能需求

### 2.1 首页看板（公开访问）

**功能描述**：未登录用户可直接访问首页，查看公开信息。

**公开内容**：
- **联赛赛程**：展示未来4周的比赛安排（时间、场地、对阵）
- **积分榜**：展示各球队战绩排名（胜/负/胜率）
- **数据排行榜**：公开的个人数据榜单
  - 打击率排行（Top 10）
  - 安打数排行（Top 10）
  - 本垒打排行（Top 10）
  - 打点排行（Top 10）
- **最近战报**：最近3场比赛的比分和简要数据

**登录入口**：
- 顶部导航栏提供"队员登录"按钮
- 点击后弹出登录框或跳转登录页

### 2.2 联赛赛程（登录后增强）

**公开功能**（无需登录）：
- 以日历/列表形式展示全部赛程
- 单场比赛信息：时间、场地、对阵双方

**登录后增强功能**：
- 显示当前报名人数
- 队员可点击"参加"或"不参加"报名
- 显示已报名队员名单
- 标记"我参加的比赛"高亮显示

### 2.2 球队展示

**功能描述**：每个球队拥有独立的展示页面。

**球队信息**：
- **队名**：球队名称（如：Joyful A队）
- **队徽**：球队Logo图片
- **球队简介**：一段介绍文字，可包括球队历史、风格、口号等
- **联盟级别**：所属联盟（大联盟/小联盟）
- **赛季战绩**：胜场/负场/胜率

**队员视角**：
- 查看自己所属球队的信息
- 查看球队成员列表
- 查看下场比赛的战术安排和个人位置

### 2.3 战术板

**功能描述**：展示球队战术安排。

**核心功能**：
- **位置安排**：显示每位队员的守备位置
- **棒次安排**：显示打击顺序
- **战术说明**：文字形式的战术要点（如：对左投策略、双杀战术暗号等）
- **布阵图**：可视化球场位置图

### 2.4 视频回放

**功能描述**：上传和观看比赛录像。

**核心功能**：
- 比赛录像上传（支持MP4格式）
- 视频播放（基础播放控制：播放、暂停、进度条、倍速）
- 精彩时刻标记（管理员可标记时间点并添加描述，如"03:15 张三本垒打"）
- 按比赛日期浏览历史录像

### 2.5 数据记录与统计

**功能描述**：记录比赛数据并生成统计。

**数据录入（管理员/队长）**：
- 简化版打席记录：棒次、姓名、位置、打席结果
- 打席结果选项：一垒安打、二垒安打、三垒安打、本垒打、三振、四坏球、飞球出局、滚地出局、双杀、牺牲打

**个人数据展示（队员）**：
- 累计数据：出赛场次、打席数、安打数、打击率、打点、得分
- 详细数据：一垒/二垒/三垒/本垒打分项统计
- 最近趋势：近5场打击率变化曲线

## 3. UI/UX 设计规范

### 3.1 整体风格

**设计原则**：
- **简洁现代**：界面干净，留白充足，信息层次清晰
- **专业运动感**：体现棒球运动的活力与团队精神
- **高质感视觉**：精致的图标、圆角设计、柔和阴影

**色彩方案**（参考 MLB 官网专业运动风格）：

| 类型 | 色值 | 用途 |
|------|------|------|
| **主色调（MLB海军蓝）** | `#041E42` | 导航栏、页脚、主标题、球队标识背景 |
| **强调色（MLB红）** | `#BF0D3E` | CTA按钮、报名按钮、重要标记、当前选中状态 |
| **背景色（浅灰蓝）** | `#F5F7FA` | 页面背景、卡片底色 |
| **卡片白** | `#FFFFFF` | 内容卡片、弹窗背景 |
| **边框灰** | `#E2E8F0` | 分割线、卡片边框、输入框边框 |
| **主文字** | `#1A202C` | 标题、主要内容文字 |
| **次文字** | `#4A5568` | 描述文字、次要信息 |
| **金色** | `#C4A35A` | 冠军标识、特殊成就、排名Top3 |

**功能状态色**：
- **成功/已报名**：`#2D8659`（深绿）
- **进行中/警告**：`#E67E22`（橙色）
- **信息/链接**：`#3182CE`（蓝色）
- **错误**：`#DC2626`（红色）

### 3.2 动效设计

**页面转场**：
- 页面切换使用淡入淡出（fade-in/out），时长200-300ms
- 列表加载使用依次滑入（stagger slide-up），营造流畅感

**交互动效**：
- 按钮悬停：轻微上浮（translateY -2px）+ 阴影加深
- 卡片悬停：微妙放大（scale 1.02）+ 阴影增强
- 点击反馈：按钮按下时缩放（scale 0.98）
- 数据更新：数字变化使用计数动画

**数据可视化动效**：
- 图表加载时从0动画增长到实际值
- 雷达图逐项绘制，增强仪式感
- 趋势图线条从左到右绘制

**加载状态**：
- 骨架屏（Skeleton Screen）替代传统loading
- 图片加载使用模糊渐显（blur-up lazy loading）

### 3.3 组件规范

**卡片设计**：
- 圆角：12-16px
- 阴影：0 4px 6px rgba(0,0,0,0.05)
- 悬停阴影：0 10px 15px rgba(0,0,0,0.1)
- 背景：纯白或轻微渐变

**按钮设计**（MLB风格）：
- **主按钮**：MLB红（#BF0D3E）填充 + 白色粗体文字 + 8px圆角，悬停时加深为 `#A00B34`
- **次按钮**：白色填充 + MLB海军蓝（#041E42）边框和文字 + 8px圆角
- **幽灵按钮**：透明背景 + 白色/主色文字，悬停时显示下划线或背景色
- **尺寸**：大按钮（48px高，用于主要CTA）、中按钮（40px高，标准操作）、小按钮（32px高，辅助操作）
- **特殊**：重要操作（如报名）使用红色填充 + 白色文字，营造紧迫感

**表单设计**：
- 输入框：底部边框或圆角矩形，聚焦时主色边框
- 标签：浮动标签（Floating Label）或顶部固定标签
- 错误提示：红色文字 + 图标，实时验证

**导航设计**（MLB风格）：
- **顶部导航栏**：固定顶部，深蓝色（#041E42）背景 + 白色文字/图标
- **Logo区域**：左侧放置 Joyful Logo，白色或反色处理
- **导航链接**：白色文字，悬停时显示红色下划线或背景高亮
- **登录按钮**：右上角红色（#BF0D3E）填充按钮，突出显示
- **移动端**：底部Tab导航，选中项使用红色高亮
- **徽标（Badge）**：红色圆点提示未读消息或待报名比赛

## 5. 设备适配规范

### 5.1 响应式断点

| 设备类型 | 断点范围 | 布局策略 |
| :--- | :--- | :--- |
| **手机** | < 640px | 单列布局，底部Tab导航，卡片全宽 |
| **平板** | 640px - 1024px | 双列布局，侧边导航或顶部导航 |
| **桌面** | > 1024px | 多列布局，固定侧边栏，最大内容区宽度1200px |

### 5.2 移动端优先设计

**触摸优化**：
- 所有可点击元素最小尺寸 44x44px
- 按钮间距至少 8px，防止误触
- 滑动操作支持（如：左滑报名/取消）

**移动端特有设计**：
- 底部固定导航栏（4-5个主要入口）
- 顶部固定标题栏（显示当前页面名）
- 下拉刷新手势支持
- 上拉加载更多

**输入优化**：
- 数字输入使用数字键盘
- 日期选择使用原生日期选择器
- 减少手动输入，多用选择器

### 5.3 桌面端增强

**大屏优化**：
- 赛程页使用日历网格视图（手机用列表）
- 数据页并排显示多个图表
- 战术板左右分栏显示（手机上下堆叠）

**键盘快捷键**（可选）：
- `ESC`：关闭弹窗
- `←/→`：切换比赛/视频

### 5.4 跨设备一致性

**状态同步**：
- 同一账号在不同设备登录，数据实时同步
- 报名操作在任一设备完成，其他设备立即更新

**性能要求**：
- 首屏加载时间 < 2秒（3G网络）
- 图片懒加载，优先加载可视区域内容
- 视频按需加载，不预加载大文件

## 6. 技术架构建议

### 6.1 前端技术栈

| 层级 | 技术选择 | 说明 |
| :--- | :--- | :--- |
| 框架 | React 18 + TypeScript | 类型安全，生态成熟 |

| 状态管理 | Zustand 或 React Query | 轻量级状态管理 |
| 构建工具 | Vite | 快速热更新，优化构建 |

### 6.2 后端技术栈

| 层级 | 技术选择 | 说明 |
| :--- | :--- | :--- |
| 框架 | Node.js + Express 或 Python + FastAPI | RESTful API |
| 数据库 | PostgreSQL | 关系型数据，支持JSON字段 |
